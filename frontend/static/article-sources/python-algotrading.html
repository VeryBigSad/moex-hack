В этом посте я собрал все известные мне полезные штуки, которые могут пригодится в алготрейдинге. Чем-то я пользуюсь сам, что-то пробовал и не мне понравилось, а чем-то не пользовался вовсе, но это не значит что это не может быть вам полезным. Не все относиться к алготрейдингу напрямую, но может быть полезным в какой-то степени.

Общие инструменты для удобства разработки

black - code formatter. Делает код единообразным, что очень удобно при просмотре git diff - сильно снижает когнитивную нагрузку.

isort - сортировка кода. Я использую только для сортировки импортов, плюсы такие же как и от black.

precommit - удобная конфигурация precommit hook’ов. Я обычно в хуки добавляю black & isort. Это спасло меня от многих багов + репозиторий всегда в чистоте.

coockiecutter - позволяет создать шаблон для проекта и переиспользовать. Очень удобно, если под каждого бота создается свой проект. Раньше я им часто пользовался. Туда же можно запихнуть конфиг для pre-commit, что очень удобно.

Библиотеки

numpy - не могу не упомянуть, must have в любом проекте.

pandas - аналогично. Этакий excel на стероидах. Must have почти в любом проекте. Если только начинаете знакомство с python, то советую изучить это библиотеку в первую очередь.

matplotib - визуализация, эквити строить и прочие красивые штуки.

seaborn - matplotlib на стероидах, чуть красивее и удобнее.

plotly - красивая визуализация.

bokeh - интерактивная визуализация. Можно фильтровать данные и график будет обновляться - модно и молодежно, почти как в JS.

asyncio - асинхронность, вебсокеты и все такое. Используется почти в любом live проекте.

gevent - удобные корутины (greenlet’ы). Библиотека сильно упрощает жизнь, если надо работать с несколькими тредами. Не очень хорошо дружит с asyncio, поэтому вместе использовать иногда сложно, но вполне возможно. Я использую в каждом втором проекте.

msgpack - как json, только гораздо быстрее. Есть реализации почти под все языки. Я использую очень часто для коммуникации между сервисами.

zmq (zeromq) - networking library. Похожа на очередь сообщений, но ей не является. Для передачи сообщений между процессами/компонентами пока ничего быстрее не встречал. Использую очень часто. Есть реализации почти под любой язык

quantlib - C++ библиотека с кучей функционала - pricing, stochastic processes, volatility estimation и все в этом духе.

TA-lib - библиотека для технического анализа, индикаторы и все такое.

scikit-learn - классические алгоритмы ML и подготовки данных.

CCXT - интеграция почти со всеми более менее ликвидными криптобиржами. Must have, если приходится торговать криптой.

pyside и pyqt5 - если нужны красивые и современные интерфейсы для десктопа (биндинги для QT)

Бектестеры

Их написано невероятное количество. Но, увы, готовых для live торговли решений мало. Зачастую каждый бектестер позволяет реализовывать какие-то определенные типы стратегий. В целом, для старта можно найти нечто подходящее и допиливать под свои нужды, благо почти все open-source.

zipline - пожалуй, самый мощный бектестер от quantopian. Местами не очень логичный в плане реализации. Есть интеграция с разными ML библитеками, хорошо дружит с pandas. Проект уже не обновляется последние пару лет.

lean - бектестер от quantconnect, написан на C#, но есть интерфейс для работы из python. Я как-то пытался им пользоваться, но почему-то не зашло. Возможно из-за ограничений IronPython (может, что-то поменялось и IronPython больше не нужен?).

VectorBT - векторизированный бектестер, больше подходит для market research. Позволяет быстро прогонять стратегии, но есть опасение что подвержен look ahead bias. Live trading не поддерживается.

backtrader - бектестер с ограниченой поддержкой живой торговли. Есть встроенная визуализация результатов и возможность создавать собственные индикаторы.

qstrader - очень качественный event-driven фреймворк с хорошей документацией и визуализацией результатов бектеста. Подходит для торговли и тестов портфельных стратегий. live торговли нет, но теоретически можно прикрутить.

RLTrader фреймворк для обучения стратегий на основе reinforcement learning. Можно использовать в качестве основы для своего RL велосипеда :)

TensorTrade - тоже фреймворк для разрабоки RL торговых алгоритмов. Выглядит намного более живым чем RLTrader. Live торговли естественно нет, только обучение.

Сервисы

blueshift - аналог quantopian, есть поддержка живой торговли, готовые датасеты. Это неплохой вариант для старта, если был опыт с подобными платформами. Есть visual programming. Интерфейс, конечно, на любителя).

quantrocket - относительно новый сервис, позволяет работать на локальной машине. Нужен docker для установки, что может стать проблемой для начинающих. Есть live trading и встроенные дата фиды.

quantconnect - сервис на основе lean фреймворка. Есть live и бектесты, позволяет делать достаточно сложные алгоритмы. Большое комьюнити и приятный интерфейс. По ощущениям - бектесты идут медленно (во всяком случае так было года 3 назад).

Утилиты для работы с портфелями

Позволяют строить оптимальные портфели, анализировать результаты и искать альфу.

Pyfolio - библиотека от quantopian для анализа портфелей и рисков

Alphalens - тоже тулза от quantopian для поиска альфы
